# Client — React Frontend

## Project Overview
React 19 SPA with Vite 7, Tanstack Router (file-based routing), Tanstack Query (server state), Socket.io-client (real-time), Tailwind CSS v4, Shadcn UI primitives.

## Commands
| Command | Action |
|---|---|
| `pnpm dev` | Vite dev server |
| `pnpm build` | `tsc -b && vite build` |
| `pnpm lint` | ESLint . |
| `pnpm preview` | Vite preview of production build |

## Route Structure
```
src/routes/
├── __root.tsx              — Root layout (sidebar, theme provider)
├── index.tsx               — Redirect to /app
├── _auth/                  — Auth-required (redirects to login if unauthenticated)
│   ├── route.tsx
│   ├── login.tsx
│   └── signup.tsx
└── app/                    — Authenticated app shell
    ├── route.tsx
    ├── conversations/
    │   ├── route.tsx
    │   ├── @me.tsx          — Direct messages index
    │   └── $conversationId.tsx — Single conversation
    └── users/
        ├── route.tsx
        └── profile.tsx
```

## Component Architecture
```
src/components/
├── ui/                     — Shadcn-style primitives (button, input, card, avatar, etc.)
├── sidebar.tsx             — Navigation sidebar
├── message.tsx             — Message bubble with reactions, receipts
├── message-input.tsx       — Compose area with emoji picker
├── direct-message-list.tsx — DM sidebar list
└── ...
src/lib/
├── queries/                — Tanstack Query definitions (auth, conversations, users)
├── mutations/              — Tanstack Query mutations (auth, conversations)
├── sockets/                — Socket.io connection + event handlers
├── hooks/                  — Custom React hooks (useDebounce, useMatchMedia)
├── context/                — React context providers (OnlineUsersProvider)
└── utils.ts                — cn() helper
```

## Code Style
- **Routing**: file-based via Tanstack Router; `routeTree.gen.ts` is auto-generated by the router plugin
- **Data fetching**: Tanstack Query in `lib/queries/` (reads) and `lib/mutations/` (writes)
- **Forms**: `@tanstack/react-form` with Zod schema validation
- **Styling**: Tailwind v4 utility classes; use `cn()` from `lib/utils.ts` for conditional class merging
- **Auth state**: Query-based (check session), not global state
- **Socket events**: handled in `lib/sockets/`
- **Theme**: `next-themes` with dark/light mode toggle
- **Imports**: `@chat/shared` for types and schemas

## Security
- JWT is HTTP-only cookie (not accessible to JS) — no client-side token storage
- `VITE_API_URL` env var — only `VITE_`-prefixed vars exposed to client bundle
- All API calls go through `lib/fetch-wrapper.ts` (handles 401 redirect to login)
