# Database Package — Prisma + PostgreSQL

## Project Overview
Prisma 7 client, Zod schemas, and browser adapter for PostgreSQL. Exports three entry points consumed by server, client, and shared packages.

## Commands
| Command | Action |
|---|---|
| `pnpm build` | `prisma generate` + `tsc` |
| `pnpm db:generate` | `prisma generate` |
| `pnpm db:migrate` | `prisma migrate dev` |
| `pnpm db:reset` | `prisma migrate reset` |
| `pnpm db:studio` | `prisma studio` |
| `pnpm dev` | `tsc -w` |

## Exports (from package.json)
| Import path | Content |
|---|---|
| `@chat/db/client` | Prisma client singleton for Node.js server |
| `@chat/db/browser` | Prisma client for browser (via `@prisma/adapter-pg`) |
| `@chat/db/schemas` | Zod schemas auto-generated from Prisma schema |

## Source Structure
```
prisma/schema.prisma       — Data model (User, Message, Conversation, etc.)
src/
├── client.ts              — Exports PrismaClient singleton
├── browser.ts             — Browser-compatible client wrapper
├── schemas.ts             — Re-exports generated Zod schemas
└── generated/             — Auto-generated (do not edit)
    ├── prisma/            — Prisma client output
    └── (zod schemas)      — prisma-zod-generator output
```

## Data Model Overview
| Model | Key fields | Relations |
|---|---|---|
| User | id, name, password (hashed), bio, imgUrl | messages, conversations, blocked users |
| Message | id, body, status (PENDING/DELIVERED/READ) | author, conversation, receipts, reactions, replies |
| MessageReceipt | readAt | message, user (unique per pair) |
| MessageReactions | emoji | message, user (unique per pair) |
| Conversation | createdAt, updatedAt | users, messages |

## Code Style
- **Schema changes**: edit `prisma/schema.prisma` only; never touch `src/generated/`
- **After schema change**: run `pnpm db:migrate` then `pnpm db:generate`
- **Zod schemas**: auto-generated by `prisma-zod-generator` — import from `@chat/db/schemas`
- **Date fields**: Prisma returns `DateTime` as JS `Date` objects

## Security
- Password field must be explicitly excluded via Prisma `select` — never use `include` on User
- No raw SQL — always use Prisma's type-safe query API
